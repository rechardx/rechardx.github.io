搭建纯静态博客的完整指南

2026-06-20

为什么不用 Hexo / Hugo？

这些工具很好，但有时候你只需要一个简单的博客，不想学习复杂的配置和主题系统。

自己搭一个的好处：

  1. 完全可控 — 每一行代码你都清楚
  2. 零依赖 — 不需要庞大的框架
  3. 学习价值 — 理解静态站点生成器的原理

架构设计

blog/
├── _posts/           # Markdown 文章
├── _templates/       # HTML 模板
├── assets/           # CSS、图片
├── build.js          # 构建脚本
└── dist/             # 输出目录（部署这个）

构建流程

  1. 扫描 _posts/ 目录下的 .md 文件
  2. 解析 front matter（标题、日期、摘要）
  3. 将 Markdown 转为 HTML
  4. 插入到模板中
  5. 输出到 dist/

部署到 GitHub Pages

# 构建
npm run build

# 推送 dist 目录到 gh-pages 分支
cd dist
git init
git add .
git commit -m "deploy"
git push -f origin main:gh-pages

就这么简单！🎉
